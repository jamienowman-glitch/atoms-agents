A. DAILY OPEN-REPO RITUAL (7 STEPS)
-----------------------------------

Every time I sit down to work on this repo, I do these in order.
If I’m tired or stressed, I still do them – even if quickly.

1. Ground in the Rules (2–5 mins)
   - Open and at least skim:
     - docs/constitution/00_CONSTITUTION.md
     - docs/constitution/01_FACTORY_RULES.md
   - Reminder to self:
     - Constitution = WHY + global rules.
     - Factory Rules = HOW work flows between Architect → Implementers → QA.
   - Optional: glance at BOSSMAN.txt to reset my head.

2. Find the Active Plan & Task (Multi²¹ or other app)
   - Open the current plan file, e.g.:
     - docs/10_MULTI21_PLAN.md  (for Multi²¹)
   - Check:
     - Which section is “Active Task”.
     - The current Task ID (e.g. M21-18) and its phases.
   - Say to myself out loud (or in notes):
     - “Today I’m working on: <TASK_ID> – <short description>.”

3. Check Logs & Status (just to know reality)
   - Open the relevant log file(s), e.g.:
     - docs/logs/MULTI21_LOG.md
   - Quick scan:
     - What was last done?
     - Did Max already mark any phases Done?
     - Is there any Claude QA stamp for recent tasks?

4. Bring the Agents Back Online (mental reset)
   - I remind myself of the four core roles:

     - Gem (Gemini) = Architect / Librarian
       - Writes and maintains plans.
       - Organises tasks into Active / Future / Completed.
       - Ensures each phase includes:
         - “Steps (Max / Implementer)”
         - “Logging & Status (Max)” instructions.
       - Does *not* implement code.

     - Max (ChatGPT Codex / implementer) = Worker
       - Only writes code and concrete docs based on Gem’s plans.
       - Must follow “Logging & Status (Max)” at the end of each phase.
       - Must update logs and move tasks between Active/Completed in plan docs.

     - Claude (Team Blue) = Clerk / QA / Health & Safety
       - Checks implementation vs plan vs Constitution vs Factory Rules.
       - Checks logs and status updates.
       - Adds a QA stamp:
         - “QA: Claude – PASS (date, notes)” or
         - “QA: Claude – FAIL (reason, required fixes)”.

     - Ossie (OSS / styling helper) = Stylist / Simple Engines
       - Helps with styling, formatting, simple engines, small OSS-flavoured tasks.
       - No big architectural decisions; follows Constitution and plans.

5. Choose ONE Micro-Phase to Focus On
   - From the Active Task, pick ONE phase or sub-step (e.g. “Phase M1: FloatingIcon Primitive”).
   - Write it somewhere visible (physical notebook, notes file, whatever):
     - “FOCUS NOW: <TASK_ID> / <PHASE>”.
   - I don’t let myself spawn extra tasks in my head until this one is either:
     - Implemented by Max and logged, or
     - Sent back to Gem to clarify.

6. Work with the Right Agent, Using the Right Template
   - If I need to update or create plans → I talk to Gem (Gemini) using the Gem snippet in Section C.
   - If I need implementation / code / concrete docs → I talk to Max using the Max snippet.
   - If I’m ready for QA → I talk to Claude using the Claude snippet.
   - If I’m designing or exploring / not repo-connected → I talk to “Control Tower” ChatGPT using the Control Tower snippet.
   - I keep reminding each agent:
     - “Respect BOSSMAN.txt and Factory Rules. Plans live in docs/*. Implementation follows plans. Logging & Status is mandatory at the end of each phase.”

7. End-of-Session Closing Ritual (2–10 mins)
   - Before I close my laptop:
     - Check the plan file for the task I worked on:
       - Did Max (or I) mark the relevant phase as Done?
       - If we finished the whole task, was it moved from Active to Completed?
     - Check the log file:
       - Is there at least one line for what got done today?
     - If QA is needed:
       - Prepare a short message for Claude (see Section C) and note “Send to Claude next session”.
   - Optional: write 2–3 bullets in a daily note:
     - What we advanced.
     - What’s blocked.
     - What’s the next micro-phase I want to tackle.
