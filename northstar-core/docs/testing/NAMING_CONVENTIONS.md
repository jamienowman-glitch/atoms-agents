# Testing Naming Conventions

All artifacts and logs generated by the test harness must follow these rules to ensure human readability and easy debugging.

## 1. Artifacts via Run Artifacts Directory
Path: `registry/run_artifacts/<test_slug>/<run_id>/`

Format:
`step_{step_index:02d}__{framework}__{agent_name}__{description}.{ext}`

Examples:
- `step_01__intern__marketing_intern__youtube_ideas.txt`
- `step_03__autogen__growth_manager__debate_round_1.txt`
- `step_06__crewai__spanish_translator__translated_list.txt`

## 2. Blackboard Snapshots
Snapshot of the blackboard state after a write operation.

Format:
`blackboard__after_step_{step_index:02d}.json`

## 3. Blackboard Keys
Keys inside the blackboard dictionary should be lower_snake_case and descriptive.

Examples:
- `youtube_ideas`
- `winning_idea`
- `spanish_titles_eu`

## 4. Constraint Compliance
- **No spaces** in filenames (use underscores).
- **Double underscore** `__` as separator between metadata fields.
